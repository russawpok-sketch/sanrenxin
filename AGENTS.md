# AGENTS.md

## 工作原则

本项目采用 AI Agent + 开发者协同开发模式。Agent 在执行任何任务时，必须遵守以下规范。

---

# 1. Git管理规范

## 开始任务前执行以下检查：

- 查看当前分支
- 拉取最新代码
- 检查工作区状态
- 确认不存在未提交变更

```bash
git status
git pull
```

---

## 完成任务后必须执行：

```bash
git add .
git commit -m "描述本次修改内容"
git push
```

要求：
- 每轮对话涉及代码修改必须提交
- Commit 信息必须明确说明修改内容
- 不允许长时间积累大量未提交代码
- 保持仓库随时可回滚

示例：
```bash
git commit -m "feat: 新增用户登录接口"
git commit -m "fix: 修复订单状态更新异常"
git commit -m "docs: 更新接口文档"
```

---

# 2. 任务执行规范

## 简单任务

满足以下条件之一：
- 修改少量代码
- 修复明显Bug
- 修改文案
- 调整配置

可直接执行。

---

## 非简单任务

满足以下任意条件：
- 涉及多个文件
- 涉及架构调整
- 涉及数据库修改
- 涉及新功能开发
- 涉及需求分析
- 涉及重构

必须严格按照以下流程执行：

### 第一步：分析任务

输出：
- 任务目标
- 实现方案
- 风险点
- 影响范围

---

### 第二步：创建Todo List

创建：
```text
docs/todo.md
```

格式：
```md
# Todo

## 当前任务
- [ ] 需求分析
- [ ] 后端开发
- [ ] 前端开发
- [ ] 测试验证
- [ ] 更新文档
- [ ] Git提交
```

---

### 第三步：分步执行

一次只完成一个步骤。完成后立即更新Todo：

```md
- [x] 需求分析
- [x] 后端开发
- [ ] 前端开发
```

---

### 第四步：校验检查

必须检查：

#### 功能检查
- 功能是否符合需求
- 是否存在遗漏

#### 代码检查
- 是否存在重复代码
- 是否存在明显逻辑问题
- 是否符合项目规范

#### 文件检查
- 是否误删文件
- 是否修改无关文件

---

### 第五步：更新文档

同步更新：
```text
docs/
```
相关文档。包括：
- 功能说明
- 接口说明
- 使用说明
- 更新记录

---

### 第六步：版本记录

文档必须维护版本号。示例：

```md
# Changelog

## v1.0.3

### Added
- 新增用户管理模块

### Fixed
- 修复登录异常

### Changed
- 优化数据库结构
```

---

### 第七步：Git提交

确认：
- Todo全部完成
- 文档已更新
- 检查通过

然后提交：
```bash
git add .
git commit -m "对应修改内容"
git push
```

---

# 3. 文件安全规范

为防止误删和覆盖：
- 不允许删除文件，除非明确要求
- 删除前必须说明原因
- 大规模修改前保留备份
- 优先增量修改而非整体重写
- 修改前先阅读相关文件

---

# 4. 代码修改原则

遵循：
- KISS（保持简单）
- DRY（避免重复）
- YAGNI（不过度设计）

禁止：
- 无意义重构
- 修改无关代码
- 未验证直接提交
- 大范围格式化导致大量Diff

---

# 5. 输出规范

Agent 回复必须包含：

## 任务分析
说明目标与方案。

## 执行进度
当前完成：
- [x] 已完成项
- [ ] 未完成项

## 修改内容
列出：
- 修改文件
- 新增文件
- 删除文件

## 校验结果
说明检查情况。

## Git状态
说明：
- 是否已提交
- Commit内容

---

# 6. 异常处理

遇到以下情况立即停止：
- 需求不明确
- 文件缺失
- 依赖异常
- 数据风险
- 大规模删除风险

先说明问题，再等待确认。

---

# 7. Agent核心原则

1. 先分析，再编码
2. 先规划，再执行
3. 先验证，再提交
4. 文档与代码同步更新
5. 每轮任务结束必须提交Git
6. 保持仓库随时可回滚
7. 不擅自删除文件
8. 非简单任务必须维护Todo List
9. 所有修改必须可追溯
10. 优先保证稳定性，其次追求速度

# 8. MVP优先原则

优先级：可运行 > 可演示 > 可扩展 > 完美架构

禁止：
- 为未来100万用户设计当前系统
- 提前优化不存在的性能问题
- 过度抽象
- 过度设计

所有功能优先完成最小可用版本（MVP）。
