# A 线：声音 AI 线 — 任务清单

> 负责人：Lead A  
> 目标：麦克风采集 → 娱乐型动物识别 → 发出变身事件 + 魔性文案  
> 技术：Web Audio API、`getUserMedia`、轻量音频特征分类

---

## 你负责什么

| 模块 | 说明 |
|------|------|
| 麦克风输入 | 权限申请、音频流采集 |
| 音频分析 | 音量、音调、节奏等特征提取 |
| 四动物识别 | 公鸡 / 猴子 / 狗 / 鸭子（模糊匹配，不要求精准） |
| 识别文案 | 「你像愤怒的鸭子」「猴子浓度100%」等 |
| 事件发布 | 识别成功后调用 `emitTransform()` 通知 B 线 |

## 你不负责什么

- 游戏画面、跑酷、碰撞（B 线）
- 四动物技能逻辑（B 线）
- 关卡地形设计（B 线）
- 前方「需要：公鸡！」提示 UI（B 线负责展示，你只管传 `animal` ID）

## Day 1：框架 + 接口

- [ ] 创建 `src/audio/` 目录结构
- [ ] 实现 `AudioCapture`：调用 `navigator.mediaDevices.getUserMedia({ audio: true })`
- [ ] 用 Web Audio API 创建 `AnalyserNode`，拿到实时音量数据
- [ ] 底部识别区：显示实时音量条 / 波形（可先占位）
- [ ] 与 B 线一起定稿 `shared/types.ts`（见 [`开发分工-共同接口.md`](./开发分工-共同接口.md)）
- [ ] 实现 `emitTransform(event)` 和 `onTransform(callback)` 事件总线
- [ ] **Mock 联调**：按键盘 `1/2/3/4` 分别模拟四种动物识别，确认 B 线能收到

---

## Day 2：识别核心

- [ ] 实现 `AudioRecorder`：每 300~500ms 切一段音频 buffer
- [ ] 实现 `FeatureExtractor`，提取以下特征：
  - [ ] 平均音量（RMS）
  - [ ] 基频区间（高/中/低）
  - [ ] 高频能量占比
  - [ ] 节奏：短促重复 vs 连续
  - [ ] 爆发感：音量突变
- [ ] 实现 `AnimalClassifier`，规则参考：

| 动物 | 特征倾向 |
|------|----------|
| 公鸡 rooster | 中高音、有节奏、连续 |
| 猴子 monkey | 高音、不规则、爆发感强 |
| 狗 dog | 中低频、短促重复 |
| 鸭子 duck | 中音、扁平、连续 |

- [ ] 输出 `{ animal, confidence }`，confidence 0~1 仅供 UI 展示
- [ ] 识别失败时返回 `null` 或低 confidence，不乱发事件

---

## Day 3：节目效果

- [ ] 实现 `RoastGenerator` 文案池，每种动物至少 5 条，例如：
  - 公鸡：「识别为暴躁公鸡」「喔喔喔浓度超标」
  - 猴子：「猴子浓度100%」「你像上树的社畜」
  - 狗：「你像加班后的狗」「汪汪力不足」
  - 鸭子：「你的鸭叫很绝望」「识别为愤怒鸭子」
- [ ] 识别失败文案：「没听出来」「再叫大声点」「这不像动物」
- [ ] 防抖：同一只动物 800ms 内不重复发事件
- [ ] 冷却：连续识别间隔 ≥ 300ms，避免误触风暴
- [ ] 底部 UI 状态：「识别成功」「猴叫失败」「你像愤怒的鸭子」

## 验收标准（A 线）

1. 浏览器能正常申请麦克风权限
2. 四种叫声（或键盘 1/2/3/4）能识别并发出 `TransformEvent`
3. 每次识别带有魔性 `roastText`
4. B 线能在 1 秒内收到事件并完成变身
5. 拒绝麦克风时有 fallback，Demo 不卡死
