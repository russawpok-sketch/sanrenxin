# B 线：游戏玩法线 — 任务清单

> 负责人：Lead B  
> 目标：横版像素跑酷 → 四动物变身 → 四技能 → 四地形 → 可玩 Demo  
> 技术：Canvas 2D（或 Phaser 3）、TypeScript

---

## 你负责什么

| 模块 | 说明 |
|------|------|
| 游戏主循环 | update / render、帧率稳定 |
| 像素角色 | 默认小人 + 四种动物外观（可先色块区分） |
| 四动物技能 | 啄墙 / 攀爬 / 跳跃 / 游泳 |
| 跑酷关卡 | 墙 / 高树 / 水池 / 深坑 |
| 游戏 UI | 左上头像、前方障碍提示、底部识别状态展示 |

## 你不负责什么

- 麦克风、音频采集（A 线）
- 动物识别算法（A 线）
- 吐槽文案生成（A 线产出，你负责展示 `roastText`）

## Day 2：四动物变身 + 技能

- [ ] 四种动物视觉区分（Demo 可用不同颜色方块 + 文字标签）：
  - [ ] rooster 公鸡 — 红
  - [ ] monkey 猴子 — 棕
  - [ ] dog 狗 — 黄
  - [ ] duck 鸭子 — 绿
- [ ] 实现 `AnimalSkills`，每种动物一个主动技能（靠近障碍时自动触发或按空格）：

| 动物 | 技能 | 触发条件 | 效果 |
|------|------|----------|------|
| 公鸡 | 啄墙 | 碰到墙 + 当前是公鸡 | 墙消失，继续跑 |
| 猴子 | 攀爬 | 碰到树/梯子 + 当前是猴子 | 沿树向上移动，越过障碍 |
| 狗 | 跳跃 | 遇到深坑 + 当前是狗 | 高跳跨越深坑 |
| 鸭子 | 游泳 | 进入水池 + 当前是鸭子 | 水中正常速度移动 |
- [ ] 错误动物惩罚（Demo 核心乐趣）：
  - [ ] 非公鸡撞墙 → 卡住 / 减速 / 弹回
  - [ ] 非猴子遇树 → 无法通过
  - [ ] 非狗遇坑 → 掉落失败
  - [ ] 非鸭子进水 → 溺水失败（重开或回到检查点）

---

## Day 3：关卡 + UI

- [ ] 实现 `Level` 地形块系统：

| 地形 | 类型标识 | 需要动物 |
|------|----------|----------|
| 障碍墙 | wall | 公鸡 |
| 高树/梯子 | tree | 猴子 |
| 水池 | water | 鸭子 |
| 深坑 | pit | 狗 |

- [ ] 做一段 **连续演示关卡**（PRD 核心节奏）：
  ```
  墙 → 水池 → 树 → 深坑 → （循环或结束）
  ```
- [ ] 前方 UI 提示：玩家接近障碍 200px 时显示「需要：公鸡！」等
- [ ] 左上：当前动物头像 / 名称
- [ ] 底部：展示 A 线传来的 `roastText` 和识别状态
- [ ] 失败 / 重开：掉坑、溺水后按 R 重开

---

## Day 4：联调 + 手感

- [ ] 接入 A 线真实识别事件，关闭 mock
- [ ] 调关卡间距：保证玩家有 1~2 秒反应时间叫出来
- [ ] 调后期节奏：连续障碍时形成「3 秒切 4 种声音」的手忙脚乱感
- [ ] 变身瞬间视觉反馈：闪一下 / 缩放 / 粒子（简单即可）
- [ ] 完整跑通：从起点到终点不卡死

## 交付文件（B 线）

```

├── GameEngine.ts     # 主循环
├── Player.ts         # 角色状态 + 变身

└── index.ts          # 对外入口

assets/               # 可选，像素图
```

## 验收标准（B 线）

1. 小人能自动跑酷，摄像机跟随
2. 收到 A 线事件后，1 秒内切换动物外观和技能
3. 四种技能能过对应地形
4. 错误动物有明确失败反馈
5. 连续关卡「墙→水→树→坑」可玩通
6. UI 显示当前动物、前方提示、A 线吐槽文案
